Challenge

All prospective customers of Smartsheet start their trials from the same place: the Smartsheet website’s trial signup form. I have built many rounds of A/B tests on this form’s UI over my time at Smartsheet, with a particular focus of not only increasing form submissions but improving the quality of leads that make their way to sales.

For this project, the goal was to see if we could ask users for less information, without sacrificing the ability to generate quality leads for sales. Marketing and Demand Generation stakeholders were interested in trying a ZoomInfo API integration, but they wanted to see how it impacted form fills and conversions into paid Smartsheet accounts.

Solution

For this A/B test, I turned the form into a two-step experience with the augmentation of data from ZoomInfo’s Form Complete API. This API gathers GDPR-compliant publicly available information to support sales teams in vetting prospective leads.

Within Optimizely, our A/B testing platform, I built the two-step experience using JavaScript, CSS, and HTML, and ntegrated the FormComplete API. With the API integrated, the second step of the form experience is dynamic: it only prompts the user for data that ZoomInfo was unable to augment. This means that for the second step of the form experience, the user would likely enter less information than on the original full contact form.

Results

This original API integration boosted form conversion rates by 22% for users submitting with non-ISP email addresses and was considered a pivotal change to our trial experience for the company.

Following this A/B test, I continued to iterate on the trial form functionality on several additional projects:

  1. Worked closely with our Principal Engineer and MarTech team to integrate the FormComplete API to into nearly all forms on the primary Smartsheet site.
  2. A/B tested an email-only form that still sends any ZoomInfo augmented data to sales, which reduced the need for user input even further.
  3. Test how the business requirement to implement different Smartsheet app regions impacts the UI/UX of the form.
  4. Currently, I'm working with UI/UX designers and numerous Smartsheet app stakeholders to A/B test the best way to present SSO options to users in lieu of the traditional form fill. The first round of testing has further increased trials by up to 29%.
