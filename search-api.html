Search API Migration

Overview

Supported the migration from Coveo to OpenSearch, ensuring preservation of critical front end functionality.

$150k annual cost savings
2 migrations completed concurrently

Smartsheet

Technologies used

Challenge

For two of Smartsheet’s primary enterprise web properties, we leveraged Coveo’s Search API for site search functionality. While it is an extremely powerful API, we realized that we were only using a small fraction of its capabilities and that the value of our actual usage was far disproportionate to its cost to the business.

Additionally, we found that using a proprietary system felt like a bit of a black box, where it was difficult to control some functionality on our site at the level we would like.

Lastly, on the front end side, we felt our approach towards theming, styles, and overrides to Coveo had become disorganized and difficult to maintain and was overdue for a clean-up.

Solution

As engineers on Drupal, an open source CMS, it felt only natural to seek out an open source solution for our search experience. Our back end engineers conducted thorough research and recommended a migration to Open Search’s Search API, an open source option with existing Drupal modules, documentation, and an open source community to lean on for issue resolution.

While our back end engineers worked to integrate the new API, my role on this project was to lead the front end rebuild to support the updated search functionality. The team did two migrations concurrently on two different enterprise websites:

Help Center Site

Included not only a migration, but included partnering closely with UI/UX designers to do a design overhaul to improve the UI/UX of facets, sorts, and results on a single search experience.

Primary Smartsheet Site

This site contained dozens of search experiences across various areas of the site, so it needed a broader approach that minimized redundant theming and templates. This effort included a consolidation of numerous repetitive Coveo templates into a centralized Drupal View template.

I made it easy to spin up new search Views that contained styled filters and consistent visual components. Because these search experiences were styled to match our global component system, I was able to integrate our existing component theming and styles into the search experience in a more efficient way than we had done previously.

Results

After completing the migration, Smartsheet was able to scale back their contract with Coveo substantially, which saves the business $150k annually.

We completed our migration on both sites without any downtime or disruption to the user experience. For both sites, we now have a simplified front end approach that is far easier to maintain and expand upon in the future.
