Challenge

When migrating Smartsheet’s websites from Drupal 7 to Drupal 8, the engineering team identified the opportunity to do an entire design system overhaul. Not only should the site look more modern and on brand, it should be modular, have a clear path for visual updates and maintenance, and a component structure that makes it easy for web authors to build pages.

Solution

We determined the best path forward was to leverage the exciting new features that were widely supported in CSS without any special framework. Flexbox, grid, and CSS custom properties could unlock all we needed for a simple variable-based approach to colors, spacing, and overall layout consistency.

Taking principles like intrinsic design we set out to build a component library where we didn’t think only about mobile versus desktop and strict concepts like “grid with exactly three items,” but about flexible elements that could adapt based on real-life content needs and the component’s container size, not just the viewport size.

We set to work collaborating with UI/UX designers to architect an atomic design system in PatternLab where we could organize everything from button styles (atoms) to grids of tiles (organisms) and use these building blocks to create templates for different content types and pages.

In Figma Dev Mode (previously in Zeplin and Invision), we have worked with the design team to ensure all global variables/tokens are mapped to all components and redlines. By using CSS custom properties, we were able to build out over ### components (and counting) that support six color variations across three different schemes (light mode, dark mode, and special events color scheme). This means every component in the entire design system can easily be used in any of these 18 different color variations without time-consuming color customizations needed at a per-component level:

After prototyping in PatternLab, these components were built as Paragraphs within Drupal. These components are built as modular standalone elements a web author can drag and drop within the Bricks UI to build endless custom layout combinations. An author can choose the section’s color variation and preset size/alignment options, but all choices map back to our established global design system.

Results

Upon initial launch of the new design system and broader site migration, Smartsheet’s primary website had a fresh new look, modern user experience, improved authoring experience, and promising metrics to go with it: the project exceeded its offset goal by +71%, with $1MM in ARR attributed to the website rebuild and migration.

Currently, we have over 10,500 organism-level components in use across the primary website.

Since its inception, we have been able to continually improve our component library to stay on brand and keep up with design standards in the industry, integrating more progressive enhancements and adding new features to components along the way.

Over my years at Smartsheet, I have grown in responsibility to take on engineering ownership and governance of our global design system and have onboarded new front end engineers into the system. We have conducted a thorough accessibility remediation of the component library to ensure it meets WCAG standards. We have since brought this design system to multiple other web properties to ensure cohesion for the Smartsheet web experience.

Future

In the long term, the goal is to continue to evolve components while minimizing design system sprawl. We regularly test component performance with approaches like heatmapping and user testing, making sure our interactive components get the engagement we’d expect and continue to find opportunities for improvement.
